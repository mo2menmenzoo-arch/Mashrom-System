# 🍄 نظام إدارة صوبة الماشروم

نظام ويب متكامل لإدارة دورات إنتاج الماشروم والمالية والمخزون والعمليات اليومية.
واجهة عربية بالكامل (RTL)، مبنية بـ Next.js 15 و Prisma و PostgreSQL.

---

## 📋 الميزات الأساسية

- **إدارة الدورات**: دورات إنتاج مدتها 60 يوماً مع تتبع تلقائي للتقدم
- **لوحة تحكم شاملة**: إيرادات، مصاريف، عهدة، صافي ربح في الوقت الحقيقي
- **المبيعات**: تسجيل الكراتين مع الخصم التلقائي من المخزن
- **المصاريف**: ربط تلقائي بين مصاريف الشراء وحركة المخزن
- **المخزن**: تنبيهات الصلاحية والحد الأدنى للمخزون
- **العهدة**: إيداع وصرف مع تحذيرات الرصيد المنخفض
- **جدول التشغيل**: قراءات يومية (حرارة، رطوبة، CO₂) مع تنبيهات الانحراف
- **التقارير**: P&L لدورة واحدة أو عدة دورات، تصدير PDF و Excel
- **الإشعارات**: Web Push يومية للتذكير بقراءات التشغيل
- **الصلاحيات**: ثلاث أدوار (مدير، مشغّل، محاسب)
- **سجل التدقيق**: متابعة كاملة لجميع التعديلات

## 🚀 البدء السريع

### 1. تثبيت الحزم

```bash
npm install
```

### 2. ضبط متغيرات البيئة

انسخ الملف `.env.example` إلى `.env.local` وعبّئ القيم:

```bash
cp .env.example .env.local
```

أهم المتغيرات:
- `DATABASE_URL` — رابط قاعدة بيانات PostgreSQL
- `AUTH_SECRET` — مفتاح سري (يُولَّد بـ `openssl rand -base64 32`)
- `VAPID_PUBLIC_KEY` + `VAPID_PRIVATE_KEY` — للإشعارات (`npx web-push generate-vapid-keys`)

### 3. تهيئة قاعدة البيانات

```bash
npm run db:push      # إنشاء الجداول
npm run db:seed      # إنشاء المستخدم المدير الافتراضي
```

بيانات الدخول الافتراضية (قابلة للتعديل عبر متغيرات `SEED_ADMIN_*`):

- **البريد**: `admin@greenhouse.local`
- **كلمة المرور**: `ChangeMe!2026`

> ⚠️ **غيّر كلمة المرور فور أول تسجيل دخول**.

### 4. تشغيل التطبيق محلياً

```bash
npm run dev
```

افتح [http://localhost:3000](http://localhost:3000).

## ☁️ النشر على Vercel

1. ارفع المشروع إلى GitHub.
2. ادخل إلى [vercel.com/new](https://vercel.com/new) واختر المستودع.
3. أضف متغيرات البيئة من `.env.example` في إعدادات المشروع.
4. اربط قاعدة بيانات Neon أو Supabase (URL في `DATABASE_URL`).
5. بعد أول Deploy، شغّل الـ seed:
   ```bash
   npx vercel env pull .env.production.local
   DATABASE_URL="..." npm run db:seed
   ```
6. ستعمل مهمة الإشعارات اليومية تلقائياً عبر Vercel Cron (الساعة 5 صباحاً UTC = 7 صباحاً القاهرة).

## 📜 القواعد الأساسية للنظام (ثابتة)

1. كل سجل يجب أن يرتبط بدورة (لا يوجد سجل يتيم).
2. المخزن سجل حركات (balance = sum of `InventoryTxn.qty`) — لا يتم تعديل الكميات مباشرة.
3. مصاريف الشراء تُنشئ تلقائياً سجل مخزون مع حركة دخول (IN) داخل نفس الـ Transaction.
4. المبيعات تُنشئ حركة خروج (OUT) من المخزن تلقائياً.
5. الدورات المغلقة لا يمكن تعديلها إلا من المدير بسبب صريح (OVERRIDE) مع تسجيلها في سجل التدقيق.
6. المبالغ تُخزّن بـ `Prisma.Decimal` وليس `Number` (حفاظاً على الدقة المالية).
7. المنطقة الزمنية: UTC في قاعدة البيانات، `Africa/Cairo` لحساب يوم الدورة.
